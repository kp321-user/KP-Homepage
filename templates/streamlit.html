Streamlit Tutorial — From Zero to a Working GUI

This tutorial walks through Streamlit step by step, building up from a blank script to a functional interface — using examples relevant to a Python beginner working on local tools (file pickers, logging, automation scripts).

1. What Streamlit Is

Streamlit turns a plain Python script into a web-page-style interface — text boxes, buttons, file uploads, sliders — without writing any HTML, CSS, or JavaScript.

You write Python top to bottom, like a normal script. Streamlit re-runs the entire script every time something on the page changes (a button click, a text input, etc.). This is the single most important thing to understand about how Streamlit behaves — more on this in Section 6.

2. Installation & Running

pip install streamlit

Run a Streamlit script differently than a normal Python file — not python app.py:

streamlit run app.py

This starts a local web server and opens your default browser to http://localhost:8501.

3. The Basics — Text and Layout

Create a file app.py:

import streamlit as st
    
    st.title("My First Streamlit App")
    st.header("This is a header")
    st.subheader("This is a subheader")
    st.write("This is regular text.")
    st.markdown("You can also use **Markdown** formatting directly.")

Run it:

streamlit run app.py

Key functions: | Function | Purpose | |—|—| | st.title() | Largest heading, usually used once at the top | | st.header() | Section heading | | st.subheader() | Smaller section heading | | st.write() | General-purpose — can print text, numbers, dataframes, even charts | | st.markdown() | Renders Markdown formatting (bold, links, etc.) |

4. Input Widgets

This is where Streamlit becomes interactive. Every input widget returns a value you can use immediately in your code.

import streamlit as st
    
    # Text input
    name = st.text_input("Enter your name")
    
    # Number input
    age = st.number_input("Enter your age", min_value=0, max_value=120)
    
    # Dropdown
    category = st.selectbox("Choose a category", ["News", "Tech", "Politics"])
    
    # Checkbox
    agree = st.checkbox("I agree to the terms")
    
    # Slider
    volume = st.slider("Volume", 0, 100, 50)  # min, max, default
    
    st.write(f"Hello {name}, you are {age} years old.")
    st.write(f"Category selected: {category}")
    st.write(f"Agreed: {agree}")
    st.write(f"Volume: {volume}")

Every time you interact with any widget, Streamlit reruns the whole script from top to bottom, with the new widget values plugged in. This is why st.write(f"Hello {name}...") updates live as you type — no manual refresh needed.

5. Buttons and Conditional Logic

Buttons work a bit differently from other widgets — they return True only for the single rerun that happens right after being clicked, then go back to False.

import streamlit as st
    
    title_input = st.text_input("Video title")
    url_input = st.text_input("YouTube URL")
    
    if st.button("Add Log Entry"):
        st.write(f"Title: {title_input}")
        st.write(f"URL: {url_input}")
        st.success("Entry added!")  # green confirmation box

Other feedback boxes:

st.success("This worked!")    # green
    st.error("Something failed")  # red
    st.warning("Be careful")      # yellow
    st.info("Just FYI")           # blue

6. Understanding Streamlit’s Rerun Behavior (Important!)

This is the concept that trips up most beginners.

Every interaction reruns your entire script from the top. Click a button → whole script runs again. Type a letter into a text box → whole script runs again.

This means: - Any variable not explicitly saved will reset to its default every rerun - If you want something to persist across reruns (like a counter, or a running log inside the session), you need st.session_state

import streamlit as st
    
    if "counter" not in st.session_state:
        st.session_state.counter = 0
    
    if st.button("Increment"):
        st.session_state.counter += 1
    
    st.write(f"Counter: {st.session_state.counter}")

Without session_state, the counter would reset to 0 every single time the button is clicked, because the script re-runs from scratch and counter = 0 would re-execute every time.

7. File Uploads

import streamlit as st
    
    uploaded_file = st.file_uploader("Upload an MP3 file", type=["mp3"])
    
    if uploaded_file is not None:
        st.write(f"File name: {uploaded_file.name}")
        st.audio(uploaded_file)  # adds a playable audio widget

Note: uploaded_file is a file-like object held in memory — if you need to save it to disk to process with another tool (e.g. Whisper), write it out first:

with open(uploaded_file.name, "wb") as f:
        f.write(uploaded_file.getbuffer())

8. Layout — Columns and Sidebar

Columns (side-by-side widgets):

col1, col2 = st.columns(2)
    
    with col1:
        st.text_input("First Name")
    
    with col2:
        st.text_input("Last Name")

Sidebar (persistent panel on the left):

st.sidebar.title("Settings")
    model_choice = st.sidebar.selectbox("Whisper model", ["tiny", "base", "small", "medium"])

9. Displaying Data

import streamlit as st
    import pandas as pd
    
    data = pd.DataFrame({
        "Title": ["Video A", "Video B"],
        "Date Added": ["2026-06-18", "2026-06-19"]
    })
    
    st.dataframe(data)        # interactive, sortable table
    st.table(data)            # static table
    st.bar_chart(data.set_index("Title"))  # quick chart

This is useful if you want to display your transcriptions_list.json log as a table directly in the GUI.

10. Custom Styling (Theme + CSS)

Theme via config file — create .streamlit/config.toml in your project folder:

[theme]
    primaryColor = "#238636"
    backgroundColor = "#0d1117"
    secondaryBackgroundColor = "#161b22"
    textColor = "#c9d1d9"
    font = "sans serif"

Custom CSS injection for finer control (font sizes, etc.):

st.markdown("""
        <style>
        html, body, [class*="css"] {
            font-size: 18px;
        }
        </style>
    """, unsafe_allow_html=True)

11. Putting It Together — A Mini Real Example

This mirrors a simplified version of a transcription logging tool:

import streamlit as st
    import json
    import os
    from datetime import datetime, timezone
    
    LOGFILE = "transcriptions_list.json"
    
    st.title("Transcription Logger")
    
    title = st.text_input("Video title")
    url = st.text_input("YouTube URL")
    
    if st.button("Add Entry"):
        if not title:
            st.error("Title is required.")
        else:
            try:
                with open(LOGFILE, "r", encoding="utf-8") as f:
                    log = json.load(f)
            except FileNotFoundError:
                log = []
    
            new_id = log[-1]["id"] + 1 if log else 1
    
            log.append({
                "id": new_id,
                "title": title,
                "url": url,
                "date_added": datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ")
            })
    
            with open(LOGFILE, "w", encoding="utf-8") as f:
                json.dump(log, f, indent=4)
    
            st.success(f"Added entry: {title}")
    
    # Show existing log as a table
    if os.path.isfile(LOGFILE):
        with open(LOGFILE, "r", encoding="utf-8") as f:
            log_data = json.load(f)
        st.subheader("Log History")
        st.dataframe(log_data)

This single script: takes input, validates it, writes to the JSON log, and displays the full history as a table — all without any HTML.
