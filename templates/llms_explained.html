Overview

This document contains a structured discussion about where LLM knowledge comes from, who owns the information, transformative-use arguments, how information becomes mathematics, statistical bias in AI systems, what an LLM physically is, and how token generation works on GPUs.

1. Sources of Information and Physical Storage

An LLM is trained on huge collections of text and other data, including public websites, books, reference material, documentation, academic papers, licensed datasets, code repositories, and conversations created by trainers.

The training process does not work like storing copies of web pages in a searchable database. Instead, the model learns statistical patterns in language.

Physical Storage

Physically, the model exists as massive numerical parameter files ("weights"), GPU memory tensors, and distributed datacenter storage. The knowledge is encoded mathematically inside floating-point tensors and neural network parameters distributed across SSD arrays, high-speed networking systems, GPU VRAM, and backup storage systems.

What Knowledge Looks Like Internally

The model does not contain explicit encyclopedia entries, symbolic fact tables, or neat databases of sentences. Instead it contains statistical relationships, semantic structure, conceptual correlations, and probability distributions.

"Paris" becomes strongly associated with "France," "capital city," and "Eiffel Tower" through distributed mathematical relationships.

Training vs Live Information

Base Training Knowledge — embedded into model weights during training.

Live External Information — may come from search engines, APIs, live webpages, or external databases.

Physical Scale

Large frontier models may involve tens of thousands of GPUs, petabytes of training data, multi-megawatt power consumption, and terabytes of parameter storage.

A search engine is like a librarian retrieving documents. An LLM is more like a brain compressing statistical patterns into neural connections.

2. Ownership of LLM Information

Ownership depends on which layer you mean.

Original Source Material

Usually owned by authors, publishers, websites, organizations, software developers, and licensing partners. An AI company does not automatically own original source material simply because it trained on it.

Model Weights

The trained neural network weights are usually owned by the company that trained the model. These weights are proprietary intellectual property unless openly released.

User Conversations

Ownership depends on platform policies, regional laws, enterprise agreements, and terms of service.

Generated Outputs

The legal status of AI-generated output is still evolving. Different jurisdictions treat copyrightability, authorship, and commercial rights differently.

Legal Distinction

Does training on copyrighted material create infringement?

Courts and regulators debate fair use, transformative use, data mining rights, compensation systems, and licensing models.

3. Public Information vs Public Domain

Public accessibility and legal ownership are different things. Something can be publicly viewable and still copyrighted, licensed, or restricted.

Publicly Accessible

Means people can access it — but the creator may still retain copyright, redistribution rights, and licensing control.

Public Domain

Means no exclusive copyright ownership remains — such as expired copyrights, some government works, or explicitly released material.

Attribution

An LLM may summarize, paraphrase, or synthesize without citing a source. That does not place the original material into the public domain.

Why This Is Controversial

Critics argue AI may economically substitute creators, creators may not be compensated, and attribution may be insufficient. Supporters argue training is transformative, models learn patterns rather than storing copies, and training resembles learning or indexing.

Memorization Risk

LLMs can sometimes regenerate memorized snippets, copyrighted passages, or code fragments — especially when content is highly duplicated or prompts strongly target the material.

4. Steelman: "Training Is Transformative"

Training transforms source material into abstract mathematical representations rather than redistributing the original works.

Human Learning Analogy

Humans can read books, absorb writing styles, learn facts, and later create original writing without violating copyright merely by learning. The argument claims LLMs are analogous.

Compression Argument

Models train on petabytes of text yet weights are vastly smaller than the source corpus. This suggests abstraction, compression, and statistical representation rather than literal storage.

Functional Transformation

Original works are transformed into gradients, tensors, vector relationships, and numerical parameters. The resulting system serves prediction, summarization, reasoning assistance, and generation — rather than direct reading consumption.

Strongest Counterargument

Critics argue training still requires copying, outputs may substitute creators, scale changes the ethical landscape, and models sometimes reproduce copyrighted material.

5. How Information Becomes Mathematics

Text becomes numbers which are transformed through large mathematical systems.

1. Tokenization

Example: "The cat sat." becomes tokens ["The", " cat", " sat", "."] with integer IDs:

"The"  -> 1045
" cat" -> 8921
" sat" -> 4410
"."    -> 13

2. Embeddings

Each token becomes a vector: "cat" → [0.182, -0.044, 1.203, ..., -0.551]

Words with related meanings become geometrically close.

3. Neural Network Layers

Core equation:

y = f(Wx + b)

Where x = input vector, W = weight matrix, b = bias, f = nonlinear function.

4. Attention Mechanism

Attention(Q,K,V) = softmax(QK^T / sqrt(d))V

This computes relationships between tokens.

5. Training via Error Correction

Weights update using gradient descent:

W_new = W_old - η∇L

6. Meaning as Geometry

Relationships emerge such as:

king - man + woman ≈ queen

Concepts become directions in vector space.

7. Where Knowledge Lives

Knowledge is distributed across billions or trillions of numerical weights rather than explicit symbolic facts.

8. Limitation

The model manipulates patterns extremely effectively but does not consciously understand mathematics.

6. Statistical Bias and New Discoveries

Statistical learning systems inherit popularity bias. LLMs learn from frequency — repeated patterns become dominant, rare patterns contribute less.

This can underrepresent new discoveries, niche findings, and frontier research even if they are correct.

Consequences

Lagging Knowledge — models may reflect older consensus.

Majority-Vote Distortion — popular narratives may dominate newer findings.

Academic Shadowing — established fields dominate the training distribution.

Mathematical Reason

The model approximates P(word | context) based on frequency, co-occurrence, and dataset weighting.

Mitigations

Modern systems use retrieval-augmented generation (RAG), live search, expert fine-tuning, and curated datasets to reduce these limitations.

Philosophical Tension

LLMs are optimized for what is commonly said. Science advances through what is newly discovered and not yet commonly repeated.

7. What an LLM Physically Is

An LLM is physically large numerical parameter files running across GPU clusters distributed through datacenters.

Hardware

Inference runs on GPUs, TPUs, and AI accelerators inside industrial datacenters. Modern models are sharded across GPUs, parallelized across servers, and coordinated through distributed systems.

Supporting Infrastructure

Datacenters include networking hardware, cooling systems, storage arrays, power infrastructure, and routing systems.

"Thinking" is large-scale matrix multiplication executed on GPUs.

An LLM is a distributed cloud-based computational system executing massive real-time tensor algebra operations.

8. One Token Generation Step on a GPU

A token generation step computes probabilities for the next token.

Input Embeddings

X ∈ ℝ^(n × d)

Linear Projections

Q = XW_Q
K = XW_K
V = XW_V

Large matrix multiplications create queries, keys, and values.

Attention Scores

A = QK^T / sqrt(d_k)

Softmax Normalization

P = softmax(A)

Weighted Mixing

Z = PV

Each token becomes a weighted combination of others.

Feedforward Network

F(x) = W_2σ(W_1x + b_1) + b_2

Residual Connections

x' = x + Attention(x)
x'' = x' + MLP(x')

Final Logits and Probability Distribution

logits = h_nW_vocab
P(token) = softmax(logits)

Example: "cat" → 0.12, "dog" → 0.09, "." → 0.30

Sampling

The next token is selected using argmax, temperature sampling, or top-k/top-p sampling.

GPU Reality

The GPU mainly performs matrix multiplications, tensor operations, exponentials, reductions, and vector additions — with thousands of parallel threads executing simultaneously.

A token generation step is a forward pass through a giant differentiable mathematical function producing a probability distribution over vocabulary space.

Final Summary

An LLM is fundamentally a statistical learning system encoded as large numerical tensors, trained on massive corpora, and executed through GPU matrix algebra. Its "knowledge" exists as distributed mathematical relationships across billions or trillions of parameters rather than explicit symbolic facts.
